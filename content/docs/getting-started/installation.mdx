title: Installation

---

Basalt ships as a set of Docker images, orchestrated with Docker Compose. A single machine can run
the entire stack (panel, backend, database and a node daemon), and you can add more nodes later.

## Prerequisites

- A 64-bit Linux host (macOS and Windows work for evaluation via Docker Desktop)
- [Docker Engine](https://docs.docker.com/engine/install/) 24+ with the Compose plugin
- 2 GB of RAM for the platform itself, plus whatever your game servers need

## 1. Create the Compose file

Create a directory for your installation and add a `docker-compose.yml`:

```yaml title="docker-compose.yml"
name: basalt

services:
  surrealdb:
    image: surrealdb/surrealdb:latest
    restart: unless-stopped
    environment:
      SURREAL_USER: basalt
      SURREAL_PASS: ${SURREAL_PASSWORD}
    volumes:
      - db_data:/usr/app/data
    command: start
    healthcheck:
      test: ["CMD", "/surreal", "is-ready"]
      interval: 1s
      timeout: 5s
      retries: 30

  backend:
    image: ghcr.io/basalt-host/backend:latest
    restart: unless-stopped
    depends_on:
      surrealdb:
        condition: service_healthy
    environment:
      SURREAL_URL: ws://surrealdb:8000
      SURREAL_USERNAME: basalt
      SURREAL_PASSWORD: ${SURREAL_PASSWORD}
      SURREAL_JWT_SECRET: ${JWT_SECRET}
      BACKUP_ENCRYPTION_KEY: ${BACKUP_ENCRYPTION_KEY}
      BASALT_PUBLIC_URL: ${PUBLIC_URL}
      BASALT_ADMIN_EMAIL: ${ADMIN_EMAIL:-}
      BASALT_ADMIN_PASSWORD: ${ADMIN_PASSWORD:-}
    ports:
      - "8080:8080"

  web:
    image: ghcr.io/basalt-host/web:latest
    restart: unless-stopped
    depends_on:
      - backend
    environment:
      BASALT_API_URL: http://backend:8080
      BASALT_JWT_SECRET: ${JWT_SECRET}
      BASALT_PUBLIC_URL: ${PUBLIC_URL}
    ports:
      - "3000:3000"

volumes:
  db_data: {}
```

<Callout type="warn" title="Image names may differ">
  Check the [releases page](https://github.com/basalt-host/basalt) for the
  current image registry and tags for your version.
</Callout>

## 2. Configure secrets

Create a `.env` file next to the Compose file:

```bash title=".env"
# Database password
SURREAL_PASSWORD=change-me

# Signs and verifies session tokens. The database, backend and web app
# must all share this value. Generate one with: openssl rand -base64 48
JWT_SECRET=change-me

# Encrypts backup archives at rest.
BACKUP_ENCRYPTION_KEY=change-me-too

# Public URL of the panel, used in links inside e-mails (invites, resets).
PUBLIC_URL=http://localhost:3000

# Optional: auto-create the owner account on first start.
# Leave unset to use the owner-creation form on first visit instead.
ADMIN_EMAIL=
ADMIN_PASSWORD=
```

<Callout type="error" title="Never ship dev defaults">
  `JWT_SECRET` signs every session in your panel and `BACKUP_ENCRYPTION_KEY`
  protects your backup archives. Set unique random values before exposing Basalt
  to a network.
</Callout>

## 3. Start the stack

```bash
docker compose up -d
```

Once the containers are healthy, open [http://localhost:3000](http://localhost:3000). You'll be
greeted by the first-run setup.

## Environment reference

Every configuration option of the backend can be set through environment variables:

| Variable                                       | Default                      | Purpose                                                                         |
| ---------------------------------------------- | ---------------------------- | ------------------------------------------------------------------------------- |
| `APPLICATION_PORT`                             | `8080`                       | Backend HTTP port.                                                              |
| `SURREAL_URL`                                  | `ws://localhost:8000`        | SurrealDB connection URL.                                                       |
| `SURREAL_NAMESPACE` / `SURREAL_DATABASE`       | `main` / `main`              | Database namespace/name.                                                        |
| `SURREAL_USERNAME` / `SURREAL_PASSWORD`        | `basalt` / `basalt`          | Database credentials.                                                           |
| `SURREAL_JWT_SECRET`                           | dev value                    | Session token signing secret; **must** match the web app's `BASALT_JWT_SECRET`. |
| `BACKUP_ENCRYPTION_KEY`                        | dev value                    | Key used to encrypt backup archives.                                            |
| `MAIL_PROVIDER`                                | `log`                        | `log` prints outgoing mail to the backend log; `resend` sends real e-mail.      |
| `RESEND_API_KEY`                               | (none)                       | API key when `MAIL_PROVIDER=resend`.                                            |
| `MAIL_FROM`                                    | `Basalt <noreply@localhost>` | From address for outgoing mail.                                                 |
| `BASALT_PUBLIC_URL`                            | `http://localhost:3000`      | Public URL of the web panel.                                                    |
| `BASALT_ADMIN_EMAIL` / `BASALT_ADMIN_PASSWORD` | (none)                       | Auto-create the owner account on a fresh database.                              |
| `CORS_ALLOWED_ORIGINS`                         | dev value                    | Origins allowed to call the API.                                                |
| `UPLOAD_MAX_BYTES`                             | `2147483648`                 | Maximum file upload size (2 GiB).                                               |

And the web panel:

| Variable            | Default                 | Purpose                                                    |
| ------------------- | ----------------------- | ---------------------------------------------------------- |
| `BASALT_API_URL`    | `http://localhost:8080` | Base URL of the backend API.                               |
| `BASALT_JWT_SECRET` | dev value               | Verifies session cookies; must equal `SURREAL_JWT_SECRET`. |
| `BASALT_PUBLIC_URL` | `http://localhost:3000` | Used to build links in outgoing e-mails.                   |
